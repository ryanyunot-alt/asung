Google Drive 연동 설정

아성빌 관리 앱 · 단계별 가이드
예상 소요: 약 10분 · 비용: 무료

🎯 이 가이드를 끝내면

본인 구글 계정으로 두 개의 키(클라이언트 ID, API 키)를 발급받게 됩니다. 이걸 앱 설정에 붙여넣으면, 데이터가 본인의 구글 드라이브에 자동으로 저장·동기화됩니다. PC와 폰에서 같은 데이터를 볼 수 있게 돼요.

1
Google Cloud 프로젝트 만들기
~ 2분

먼저 본인 구글 계정으로 "프로젝트"라는 작업 공간을 만들어야 합니다. 무료이고 다른 사람은 못 봅니다.

  1. 아래 버튼을 눌러 Google Cloud Console로 이동
    → 새 프로젝트 만들기
  2. 본인 구글 계정으로 로그인 (이미 되어있을 수도 있음)
  3. 화면이 처음 뜨면 국가를 "대한민국"으로 선택하고 서비스 약관 동의 후 진행
  4. 프로젝트 이름에 "asung-building" 입력 (다른 이름도 OK)
    조직 항목은 그대로 두세요 ("조직 없음")
  5. "만들기" 버튼 클릭
  6. 알림 종 아이콘에서 "프로젝트 만들기 완료" 메시지가 뜨면 클릭 → 프로젝트 선택됨
💡 상단에 "asung-building"이라고 표시되면 성공입니다.
2
Google Drive API 활성화
~ 1분

방금 만든 프로젝트가 드라이브에 접근할 수 있도록 권한을 켜주는 과정입니다.

  1. 아래 버튼 클릭
    → Google Drive API 페이지
  2. 상단에 본인 프로젝트("asung-building")가 선택되어 있는지 확인
  3. 파란색 "사용" 또는 "Enable" 버튼 클릭
  4. 잠시 기다리면 활성화됨 ✓
💡 다른 프로젝트가 보이면? 화면 상단의 프로젝트 선택 드롭다운에서 "asung-building"을 선택하세요.
3
OAuth 동의 화면 설정
~ 3분

"이 앱이 본인 드라이브에 접근해도 되나요?"라고 물어볼 안내 화면을 설정합니다.

  1. 아래 버튼 클릭
    → Google Auth Platform
  2. "시작하기 (Get Started)" 버튼 클릭
  3. 앱 정보 입력:
    앱 이름
    아성빌 관리
    사용자 지원 이메일
    본인 구글 이메일 선택
    "다음" 클릭
  4. 대상 (Audience): "외부 (External)" 선택 → 다음
    개인 구글 계정은 "외부"만 선택 가능. 본인만 사용할 거니까 문제없습니다.
  5. 연락처 정보: 본인 이메일 입력 → 다음
  6. 완료: "User Data Policy에 동의" 체크 → "계속" → "만들기"

📌 본인을 테스트 사용자로 등록 (중요!)

  1. 왼쪽 메뉴에서 "대상 (Audience)" 클릭
  2. "테스트 사용자" 섹션의 "+ 사용자 추가" 클릭
  3. 본인 구글 이메일 입력 → 저장
⚠️ 이 단계를 빠뜨리면 안 됩니다! 본인이 테스트 사용자로 등록되어 있어야 앱에서 본인 드라이브에 접근할 수 있습니다.
4
API 키 발급 ① — Client ID
~ 2분

드디어 첫 번째 키를 만듭니다. 이게 앱을 식별하는 ID 역할을 해요.

  1. 아래 버튼 클릭
    → 클라이언트 페이지
  2. "+ 클라이언트 만들기 (Create Client)" 클릭
  3. 애플리케이션 유형: "웹 애플리케이션 (Web application)" 선택
  4. 이름: "asung-app" (또는 원하는 이름)
  5. 승인된 자바스크립트 출처 (Authorized JavaScript origins):

    앱을 사용할 주소를 등록해야 합니다. "+ URI 추가" 버튼으로 아래 주소들을 추가:

    로컬 파일에서 사용 http://localhost
    로컬 파일에서 사용 (필수) null
    💡 "null"은 정확히 그대로 입력하세요. (HTML 파일을 폰/PC에 직접 저장해서 열 때 필요합니다)
    💡 만약 나중에 GitHub Pages 같은 곳에 올리실 거면 그 주소도 추가 (예: https://username.github.io)
  6. "만들기" 클릭
  7. 팝업이 뜨면 "클라이언트 ID"를 복사 해서 메모장에 붙여넣어 두세요!
    생성된 클라이언트 ID 예시:
    123456789012-abc...xyz.apps.googleusercontent.com
✅ 첫 번째 키 발급 완료!
나중에 복사 못 해도 클라이언트 페이지에 다시 들어가서 확인할 수 있습니다.
5
API 키 발급 ② — API Key
~ 1분

두 번째 키입니다. 이건 API 호출 시 인증용으로 쓰여요.

  1. 아래 버튼 클릭
    → 사용자 인증 정보 페이지
  2. 상단의 "+ 사용자 인증 정보 만들기" 클릭
  3. 드롭다운에서 "API 키" 선택
  4. 자동으로 키가 생성됩니다. 표시된 키를 복사해서 메모장에 붙여넣어 두세요!
    생성된 API 키 예시:
    AIzaSyA1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6Q
  5. "닫기" 클릭
💡 키 보안: "키 제한" 설정은 지금은 하지 않아도 됩니다. 나중에 더 안전하게 만들고 싶으면 "HTTP 리퍼러 제한"으로 본인이 사용할 도메인만 허용하도록 설정할 수 있어요.
6
아성빌 앱에 키 입력하기
~ 1분

이제 발급받은 두 개의 키를 앱에 붙여넣기만 하면 끝!

  1. 아성빌 관리.html 파일 열기 (새 버전)
  2. 상단 메뉴에서 "설정" 탭 클릭
  3. "Google Drive 연동" 섹션에서:
    • 📌 Client ID 칸에 STEP 4에서 복사한 값 붙여넣기
    • 📌 API Key 칸에 STEP 5에서 복사한 값 붙여넣기
  4. "저장하고 연결" 버튼 클릭
  5. Google 로그인 팝업이 뜸 → 본인 구글 계정 선택
  6. "asung-app이(가) 다음 작업을 원합니다..." 화면이 뜨면:
    • 먼저 "고급" 또는 "Advanced" 클릭 (테스트 앱이라 경고 표시됨)
    • "asung-app(안전하지 않은 페이지)로 이동" 클릭
    • 드라이브 접근 권한 "허용" 클릭
    💡 본인이 만든 본인 앱이라 "안전하지 않음" 경고가 떠도 안전합니다. (Google 검토를 받지 않은 개인 앱은 자동으로 이런 경고가 떠요)
  7. "연결됨 ✓" 메시지가 뜨면 완료!

🎉 모든 설정 완료!

이제 자동으로 작동합니다:

  • ✅ 데이터를 변경하면 자동으로 드라이브에 업로드
  • ✅ 다른 기기에서 앱을 열면 자동으로 최신 데이터 다운로드
  • ✅ 드라이브에 아성빌_데이터.json 파일로 저장됨
  • ✅ 본인 외에는 아무도 접근 못 함 (구글 계정 보안)

✅ 완료 체크리스트

?
문제 해결

"앱이 차단되었습니다 (Access blocked)" 오류

→ STEP 3의 테스트 사용자 등록이 안 된 경우입니다. Google Auth Platform → 대상(Audience) → 테스트 사용자에 본인 이메일이 있는지 확인하세요.

"origin mismatch" 또는 "리디렉션 URI 오류"

→ STEP 4의 자바스크립트 출처에 "null"을 빼먹은 경우. 클라이언트 ID 편집해서 추가하세요.

"이 앱은 Google에서 인증하지 않았습니다" 경고

→ 정상입니다. 본인이 만든 본인 앱이라서 그래요. "고급" → "이동"으로 진행하시면 됩니다.

7일마다 로그인 풀림

→ 테스트 모드의 특성입니다. 다시 로그인하면 계속 사용 가능. 더 길게 유지하려면 Auth Platform에서 "프로덕션으로 게시"하면 되지만, 개인 사용엔 테스트 모드로 충분합니다.
